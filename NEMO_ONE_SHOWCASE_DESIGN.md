# Nemo One 官网介绍页面设计与开发交接文档

## 1. 文档用途

本文件用于把 Nemo One 首页产品介绍区完整交接给下一次 Codex 对话。

下一位 Agent 应先阅读项目根目录的 `AGENTS.md`，然后阅读本文件，只检查本任务涉及的前端文件。除非用户后续明确提出，不要处理仓库内其他页面、后端、路由或已有未提交改动。

## 2. 用户目标

在 DAIL Tech 公司官网首页加入一段独立的 Nemo One 产品介绍区。

这个区块的重点不是堆砌参数，而是用简洁文案和具有视觉冲击力的产品旋转动画，让访问者快速形成以下认知：

1. Nemo One 是一台面向医疗现场的 AI 一体机。
2. 它由本地 AI 硬件和 Nemo AIOS 组成。
3. 产品具有实体设备的可信感、科技感和高端工业设计感。

用户强调动画质量高于文字设计。整体应接近高端硬件官网的产品电影，而不是普通轮播图或电商图片切换。

## 3. 已确定的设计方向

- 背景：纯黑，避免复杂纹理和大面积渐变。
- 文字：白色或低饱和灰白色，排版简洁、克制。
- 主视觉：Nemo One 主机本体，占据画面主要面积。
- 交互：页面滚动驱动产品“转圈”过程。
- 动画参考：用户提供的眼镜视频中“正面建立产品印象 → 镜头推进 → 产品转向 → 细节特写 → 拉远并回正”的节奏。
- 实现方式：不用实时三维模型，不做预渲染，不安装建模软件；使用现有多角度产品图配合 CSS 和 JavaScript 模拟电影式旋转。
- 技术约束：优先原生 Vue、CSS 和 `requestAnimationFrame`，不要为了这一段新增 Three.js、GSAP 等依赖。

这里的“转圈”不要求产品几何体真正连续旋转 360 度。最终观感来自多角度图交叉切换、镜头缩放、轻微位移、高光扫过、蓝色内部光源和缓动节奏的组合。

## 7. 推荐画面结构

桌面端使用一个约 `240svh` 到 `280svh` 高的滚动容器，内部放置 `position: sticky`、高度为 `100svh` 的舞台。

舞台结构：

```text
┌──────────────────────────────────────────────────────────┐
│ NEMO ONE / MEDICAL EDGE                                  │
│                                                          │
│ 医疗 AI，                        主机产品                 │
│ 运行在医疗现场。                 大尺寸旋转区域           │
│                                                          │
│ 本地 AI 硬件与 Nemo AIOS...                              │
│                                                          │
│ 01 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 06   SCROLL TO ORBIT     │
└──────────────────────────────────────────────────────────┘
```

- 文字位于左侧，宽度控制在画面约 30% 到 38%。
- 产品位于中间偏右，可占画面宽度约 60% 到 72%。
- 产品不要被一个可见卡片或边框包裹，应像悬浮在纯黑空间中。
- 可以保留非常克制的蓝色底部环境光，但不要把背景做成蓝色。
- 文字不应跟随每一张图片跳动，只需在开场轻微上浮显现，随后保持稳定。

## 8. 滚动动画分镜

把整个区块滚动进度标准化为 `0` 到 `1`。

### 阶段 A：建立产品，进度 0.00–0.14

- 显示 `nemo-front.png`。
- 产品从约 `scale(0.84)` 缓慢靠近至 `scale(0.94)`。
- 文字从下方约 24px 上浮，透明度从 0 到 1。
- 蓝色内部灯光轻微增强。

### 阶段 B：开始转向，进度 0.14–0.32

- `nemo-front.png` 与 `nemo-orbit.png` 交叉淡化。
- 产品整体继续推进至约 `scale(1.05)`。
- 添加很轻的横向位移和不超过 1 度的 `rotateZ`，制造镜头运动感。
- 切换中点允许短暂出现约 1–2px 的方向性模糊，但不能长时间模糊。

### 阶段 C：高位绕行，进度 0.32–0.48

- 从 `nemo-orbit.png` 过渡到 `nemo-top.webp`。
- 产品达到最大主体尺寸，约 `scale(1.10)` 到 `scale(1.16)`。
- 一道柔和高光从机身左上扫向右下。
- 顶部透明窗和蓝色内部结构应成为视觉中心。

### 阶段 D：细节特写，进度 0.48–0.66

- 从 `nemo-top.webp` 过渡到 `nemo-macro.png`，随后切到 `nemo-cooling.png`。
- 这是全段最靠近镜头的部分。
- 可以让文字透明度降到约 0.6，避免和产品细节抢视觉中心，但不要完全消失。
- 散热孔近景停留时间不能太长，否则会让用户误以为动画已结束。

### 阶段 E：旋转最深处，进度 0.66–0.82

- 从细节图切到 `nemo-service.webp`。
- 产品开始略微拉远。
- 运动方向反转，为回到正面做准备。

### 阶段 F：回到英雄视角，进度 0.82–1.00

- 从 `nemo-service.webp` 平滑回到 `nemo-front.png`。
- 产品回到约 `scale(0.88)` 到 `scale(0.94)`。
- 文字恢复 100% 不透明。
- 最后一帧要稳定、清晰，方便自然进入下一个白色 CTA 区域。

## 9. 动画实现建议

### 9.1 图片叠层

所有角度图放在同一个绝对定位容器里，通过滚动进度计算每一层的透明度。不要在切换时直接更换单个 `<img>` 的 `src`，否则容易闪烁。

推荐关键帧锚点：

```js
const anchors = [0, 0.16, 0.32, 0.46, 0.60, 0.76, 1]
```

推荐图层顺序：

```js
[
  'nemo-front.png',
  'nemo-orbit.png',
  'nemo-top.webp',
  'nemo-macro.png',
  'nemo-cooling.png',
  'nemo-service.webp',
  'nemo-front.png',
]
```

在相邻锚点之间做交叉淡化。任意时刻最多让相邻两层明显可见，避免三张不同透视图同时叠加造成重影。

### 9.2 滚动平滑

- `scroll` 事件只更新目标进度，不直接做大量 DOM 写入。
- 使用 `requestAnimationFrame` 将当前进度缓动追向目标进度。
- 可采用约 `current += (target - current) * 0.10` 的阻尼。
- 所有视觉值尽量写入 CSS 自定义变量，例如 `--progress`、`--scale`、`--shift-x`、`--glow-opacity`。
- 被动画的元素优先使用 `transform` 和 `opacity`，避免频繁改变布局属性。

### 9.3 高光和空间感

可添加两层纯 CSS 效果：

1. 产品背后的低透明度蓝色径向光晕。
2. 产品上方使用伪元素制作的斜向高光带，在阶段 C 扫过一次。

效果必须克制。高光只用于强调金属边缘和顶部透明结构，不能覆盖产品细节。

### 9.4 鼠标视差

桌面端可以根据鼠标位置增加不超过 8px 的轻微偏移；触摸设备不启用。鼠标视差只是辅助，不能改变由滚动决定的主动画进度。

## 10. 性能要求

- 第一张产品图可以立即加载，其余图片在区块距离视口约一屏时预加载。
- 使用 `IntersectionObserver` 开始预加载并启动必要的监听。
- 图片元素添加明确的尺寸约束，避免加载后发生布局跳动。
- 不要把六张原图同时作为 CSS 背景重复创建。
- 组件离开页面或卸载时移除 `scroll`、`pointermove`、`resize` 监听，并取消动画帧。
- 目标是在常见桌面浏览器中保持顺滑；滚动过程中不得出现白屏或明显闪图。

## 11. 移动端设计

以项目现有 `900px` 断点为准。

- 区块高度可缩短至约 `180svh` 到 `210svh`。
- 文字放在屏幕上部，产品放在下半部。
- 主标题控制为两行，不要压到产品上。
- 产品容器可超出左右屏幕少量宽度，以保留设备的视觉体积。
- 减少鼠标视差、模糊和高光强度。
- 不要把动画退化为横向滑动轮播。

## 12. 无障碍与降级

- 产品舞台提供一个清晰的 `aria-label`，例如“旋转展示 Nemo One 医疗 AI 一体机”。
- 叠层中的重复图片使用空 `alt`，避免屏幕阅读器重复朗读。
- 在 `prefers-reduced-motion: reduce` 下：
  - 区块改为约 `100svh`。
  - 固定显示 `nemo-front.png`。
  - 禁用滚动驱动切换、视差、扫光和模糊。
  - 文案直接显示。

## 13. 导航栏处理

Nemo One 区块为纯黑背景。首页固定导航进入该区块时如果仍保持白色背景，会破坏沉浸感。

推荐方案是在区块活跃时给 `body` 添加一个临时类，例如：

```text
nemo-showcase-active
```

再让导航在这个状态下使用黑色或半透明黑色背景和白色文字。组件离开或卸载时必须清理该类。

不要为了这一点重写 `NavBar.vue`。当前导航文件已有用户未提交改动，优先通过新组件中的全局状态样式或极小范围兼容样式完成。

移动端菜单展开后仍应保持白色菜单面板和深色文字，只有未展开的顶部栏使用黑色外观。

## 16. 验收标准

完成后至少满足以下条件：

1. 首页 Technology 区块之后出现完整的纯黑 Nemo One 产品介绍区。
2. 文案数量少、层级清楚，没有未经确认的产品数据。
3. 滚动时能够明显感知主机由正面向其他角度转动，并在结尾回到正面。
4. 动画切图没有白闪、破图、突跳或三层严重重影。
5. 动画节奏包含推进、转向、特写、拉远，而不是简单等速淡入淡出。
6. 桌面端主机视觉占比足够大，黑色背景保持干净。
7. 手机端文字不遮挡主机，页面没有横向滚动条。
8. 减少动画模式下可以正常阅读并看到静态主机。
9. 不新增不必要的前端依赖。
10. `npm run build` 成功。
11. `git diff --check` 无空白错误。
12. 最终汇报明确列出修改文件、构建结果以及多角度图片模拟并非真实连续 3D 的限制。
